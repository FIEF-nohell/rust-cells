# Plans

Implementation plans, one file per task. Filename format: `YYYY-MM-DD-<short-slug>.md`. Each plan is stateful: it carries `status:` frontmatter (`in-progress` / `done` / `abandoned`), a milestone + checkbox body, and an append-only Log. The implementer ticks boxes live as work lands. Format and execution protocol are defined in `../rules/plan-execution.md`.
